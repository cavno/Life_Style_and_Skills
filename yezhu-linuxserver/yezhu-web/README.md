# 业主之家 · 成立业委会（Web 版）

把原微信小程序改成了**纯网页版**，绕开微信小程序的平台审核与个人 AppID 类目限制。功能与小程序版一致：**业主联名为核心**，外加成立业委会全流程指南、资格自测、换物业指南、名词解释、官方文书库，以及发起人侧的审核、导出、参数设置。

> 成立业主委员会、收集业主联名是《深圳经济特区物业管理条例》《龙岗区业主大会及业主委员会工作指导手册》（龙岗区住建局，2023年12月）明确保障的合法自治权利，整个流程的终点是把材料交给街道办事处（政府）。

---

## 一、为什么是「静态前端 + 自建后端」

业主用的是没翻墙的普通手机，**后端必须在大陆能稳定访问**，这决定了架构：

- **前端**：纯静态（HTML/CSS/JS，无构建步骤），可托管在任意静态平台。
- **后端**：一个你完全掌控的轻量 Node 服务（单文件、JSON 存储、无需数据库），没有任何第三方内容审核，最贴合「自治、免审查」。

### 访问与备案（务必先读）
- **海外 / 香港主机**（Cloudflare Pages、Vercel、Netlify、GitHub Pages，或香港 VPS）：**免 ICP 备案**，大陆可访问，但速度看线路，偶有波动。
- **大陆主机**：速度快，但公网域名**需要 ICP 备案**（约 1–2 周）。
- 折中建议：前端放 Cloudflare Pages（大陆可达、免备案），后端放一台**香港轻量服务器**。若已有备案域名，全部上大陆主机最快。

## 二、目录结构

```
yezhu-web/
├─ web/                      # 前端（部署到静态托管）
│  ├─ index.html
│  ├─ css/app.css
│  └─ js/
│     ├─ config.js           # ★ 填后端地址的地方
│     ├─ content.js          # 手册内容（流程/清单/名词/文书库）
│     ├─ ui.js               # toast/弹层/工具
│     ├─ api.js              # 数据层（HTTP / 本地预览 双模式）
│     └─ app.js              # 路由 + 全部页面
└─ server/                   # 后端（自托管）
   ├─ server.js              # 单文件服务（含全部接口）
   ├─ package.json
   ├─ .env.example
   └─ .gitignore
```

## 三、两种部署方式

### 方式 A：一体化（最简单，单域名，无跨域）
让 Node 服务同时托管前端：
1. 装 Node ≥ 18 的服务器上传整个 `yezhu-web/`。
2. 进 `server/`：`cp .env.example .env`，把 `ADMIN_PASSCODE` 改成你的发起人口令，并设 `SERVE_WEB=1`。
3. `npm install && node server.js`（建议用 `pm2 start server.js --name yezhu` 常驻）。
4. 把前端 `web/js/config.js` 的 `API_BASE_URL` 填成你的公网地址（如 `https://yz.example.com`）。
5. 浏览器打开 `https://yz.example.com/` 即是业主页面。

### 方式 B：前后端分离（前端走 CDN，访问更稳）
1. **后端**：在服务器 `server/` 下 `cp .env.example .env`（设口令，`SERVE_WEB=0`），`npm install && node server.js`，用 Nginx 反代到 HTTPS 域名（如 `https://api.example.com`）。
2. **前端**：把 `web/js/config.js` 的 `API_BASE_URL` 填成 `https://api.example.com`，将 `web/` 整个目录部署到 Cloudflare Pages / Vercel / Netlify / GitHub Pages。
3. 后端已开启 CORS，跨域可用。

> Nginx 反代要点：`client_max_body_size 15m;`（证件照片上传），并把 `/api/` 与上传转发到 Node 端口。

## 四、先本地点一遍（无需后端）

直接用浏览器打开 `web/index.html`（或 `npx serve web`）。此时 `config.js` 的 `API_BASE_URL` 为空，进入**本地预览模式**：所有数据存在当前浏览器，能把每个页面、联名表单、手写签名、导出都点一遍，方便给业主演示。**预览模式不能跨业主汇总，正式使用必须部署后端并填地址。**

## 五、身份与真实性模型（和小程序版的差异）

网页没有微信 openid，所以：

- **业主端免登录**：降低参与门槛。提交后本地存一个「编辑令牌」，凭它找回 / 修改 / 撤回自己的联名（换设备或清缓存会丢，属正常）。
- **发起人端口令登录**：用 `ADMIN_PASSCODE` 登录后，才能审核、导出、改参数。登录态是有效期 30 天的 token。
- **真实性靠审核**：和小程序版完全一致——联名人须上传**产权证 + 身份证**，发起人逐条核验后「通过」才计入分母。这是材料被街道办采信的关键。
- **房号去重**：不同提交填了相同楼栋房号，审核列表与详情会标红「房号重复」。
- **隐私**：产权证、身份证、手机号、签名图仅发起人凭 token 可读（`/api/admin/file/...` 鉴权，未登录返回 401）；普通业主只能看到自己的记录与全局进度数字。

发起人口令：在 `.env` 设 `ADMIN_PASSCODE` 即可；若没设，第一次在网页「发起人登录」里输入的口令会被设为口令（仅适合你本人先行初始化）。

## 六、上线后操作顺序

1. 部署好后端、前端，确认手机能打开页面、`/api/stats` 正常。
2. 「我的 → 发起人登录」，用口令登录。
3. 「小区参数」填：小区名、街道办、社区、人数口径分母、建筑面积合计、联名事项、当前阶段、联系人。
4. 把网址（建议生成二维码）发到业主群，业主自助「联名签字」。
5. 你在「审核」里逐条核验证件后通过；进度实时更新。
6. 达标后「导出」：复制申请书、下载联名表 CSV、或「打印 / 存为 PDF」，连同证件照片报送街道办。
7. 随进展在「小区参数」更新「当前阶段」，首页与指南的路线图会同步高亮。

## 七、两条 20% 口径

- 人数口径 = 已通过联名户数 ÷ 业主总人数（或户数）
- 投票权口径 = 已通过联名建筑面积合计 ÷ 全体业主专有部分建筑面积合计
- **达标 = 任意一条 ≥ 20%**（对应条例「或者」）。投票权数即专有部分建筑面积（㎡）。

## 八、合规与数据要点（与小程序版一致）

- 身份证、产权证属敏感材料，大规模收集前建议先与街道办确认其认可的收集方式与口径。
- 「业主总人数」与「业主总户数」的认定口径各街道可能不同，以你所属街道为准（参数页可切换）。
- 电子签名是否被直接采信由街道办掌握；导出同时支持文本复制、CSV 下载、打印 PDF，纸质 / 电子两条路都留好。
- 数据备份：备份 `server/data/db.json` 与 `server/uploads/` 即可（已在 .gitignore 中，不会误传到仓库）。

## 九、技术说明

- 后端依赖均为纯 JS（express / cors / multer / jsonwebtoken / bcryptjs），`npm install` 无需编译，便于在任意 VPS 部署。
- 数据为单文件 JSON 原子写入，适合单个小区规模；如需更高并发或多小区，可平滑换成 SQLite / Postgres（接口契约不变）。
- 接口契约：前端 `js/api.js` 与后端 `server.js` 路由一一对应，已通过启动 + 全链路冒烟测试（提交 / 找回 / 审核 / 统计 / 导出 / 鉴权 / 撤回 / 去重）。
