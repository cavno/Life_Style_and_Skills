# 置换决策 · 房贷计算(微信小程序)

把"房价 → 月供 → 所需妻子收入 → 留存现金"这张表做成了可拖动的交互工具,并入房贷计算器,整体改造为微信小程序。**纯本地计算,零网络请求**——不需要服务器、不需要配置域名白名单、不上传任何数据。

---

## 一、功能清单:保留了什么,删了什么

**新增(主页签"置换决策"):**
- 两个主滑杆:**房产总价**(300–600 万)与**妻子税后月收入**(1.5–5.0 万/月),拖动即实时重算
- **判定带**:舒适区 / 临界 / 依赖投资 / 越线 / 现金不足,五档实时变色,附月结余或缺口金额
- 价格滑杆下的**区带标尺**:绿 ≤450(建议区)/ 琥珀 450–500(稳健上限)/ 红 >500(红线),黑色三角为当前位置
- 六格指标:月供合计(公积金/商贷拆分)、月供÷收入占比、一次性支出(首付+契税+杂费)、留存人民币、安全垫可撑月数、可投资金
- **两条路径对比卡**:A 保留商贷(最低首付,绛红)vs B 只贷公积金(青蓝),各自的首付/一次性/月供/留存/垫撑月数,加 r* 滑杆与净现值判定
- 可折叠**参数区**:公积金额度(114 仅妻子 / 209 家庭 / 260 待核实,三档预设+手动输入)、可用资金、两项利率、首付比例、年限、生活支出、安全垫目标、杂费、>140㎡ 契税开关

**保留(页签"贷款测算"):**
- 组合贷计算:总价、首付比例、公积金额度、双利率、年限
- 等额本息 / 等额本金切换,含首月/递减/末月与两方式总利息对比

**删减(与本次置换决策无关):**
- ~~当前状态~~(跟踪旧贷款用,旧房将出售)
- ~~还款明细 360 行表~~
- ~~提前还款试算~~
- ~~LPR 利率历史~~

## 三、三步在电脑上跑起来

1. **装工具**:下载安装[微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)(稳定版即可)。
2. **导入**:打开工具 → 「导入项目」→ 目录选择本文件夹(`fangdai-juece/`)→ AppID 选「**测试号**」(或填你自己的小程序 AppID)→ 确定。
3. **跑**:编译完成即可在模拟器里拖滑杆。本程序无任何网络请求,**不需要**勾选"不校验合法域名",也不需要任何后端。

## 四、真机预览与发布

- **真机预览**:工具栏「预览」生成二维码,微信扫码即可在手机上用(测试号也可以预览)。
- **正式发布**(可选):需要一个已注册的小程序 AppID(个人主体可注册,选"工具"类目)。把 `project.config.json` 里的 `"appid": "touristappid"` 换成你的 AppID → 工具栏「上传」→ 在 [mp.weixin.qq.com](https://mp.weixin.qq.com) 后台设为体验版,或提交审核后正式发布。
- 纯工具类、无登录、无网络,审核一般无障碍。

## 五、默认参数(即推荐方案)与修改位置

所有默认值集中在 `pages/index/index.js` 顶部的 `DEFAULTS`,逐项有注释:

| 参数 | 默认 | 含义 |
|---|---|---|
| price | 420 | 房产总价(万),建议区中点 |
| wife | 3.0 | 妻子税后月收入(万/月) |
| available | 380 | 可用人民币 = 现金 350 + 卖旧净得约 30 |
| fundQuota | 209 | 公积金额度:110 万基数 ×(1+首套 40%+二孩 50%);114=妻子单独;260=待核实新政 |
| fundRate / commRate | 2.6 / 3.05 | 公积金 / 商贷年利率 %(商贷 = LPR3.5−45BP,以放款行为准) |
| downPct / years | 20 / 30 | 首付比例下限 % / 年限 |
| living | 1.0 | 非房月生活支出(万/月) |
| cushionT | 80 | 安全垫目标(万),≈33 个月零收入支出 |
| misc | 3 | 中介评估杂费(万) |
| rStar | 4.0 | 境内再投资年化收益率 %,驱动路径 A/B 净现值比较 |

界面"参数"折叠区可随时改,改 `DEFAULTS` 则成为新的打开默认值。

## 六、计算口径(可审计)

- **月供**:等额本息,公式与 Excel `PMT` 完全一致。回归锚点:`PMT(3.3%/12, 270, 1760452.12) = 9246.2109633523`,本工程引擎输出 `9246.2109633522`,13 位有效数字一致;中途不取整。
- **等额本金**:首月 = P/n + P·r/12,逐月递减 (P/n)·(r/12),总利息 = P·(r/12)·(n+1)/2(解析式,非循环累加)。
- **契税**:首套 ≤140㎡ 按 1%,>140㎡ 按 1.5%(开关切换)。
- **判定阈值**:月结余 = 妻子税后 − 月供 − 生活支出。≥3000 元为舒适区;0–3000 为临界;<0 为依赖投资;月供 > 收入为越线;一次性支出 > 可用资金为现金不足。
- **两条路径净现值**:`NPV(A−B) = 商贷金额 − PV(商贷月供流 @ r*/12)`。r* 高于商贷利率(3.05%)→ A(保留商贷)占优;低于 → B(砍商贷)占优;盈亏临界点恰为商贷利率本身。默认 r*=4% 时 A 领先约 +14.1 万(420 万总价、商贷 127 万口径)。
- **区带标尺**是此前分析的静态结论(按收入区间下沿);**判定带**按你实际拖入的数字实时计算,两层并存、互为校验。

**自检**:装有 node 的机器上,在 `test/` 目录运行 `node page_test.js`,20 项断言(默认值、阈值切换、契税、预设、非法输入、两方式切换等)应全部 PASS。
