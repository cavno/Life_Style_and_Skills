# 业主之家 · 成立业委会 微信小程序（v2）

以**小区业主**的视角，覆盖"从联名到选出业委会、再到换物业"的完整路径。**核心功能仍是业主联名**（线上替代挨家挨户签字），其余功能全部围绕《龙岗区业主大会及业主委员会工作指导手册》设计。

法规依据：《深圳经济特区物业管理条例》《龙岗区业主大会及业主委员会工作指导手册》（龙岗区住房和建设局，2023年12月）。

---

## 一、相比 v1 的变化（已对齐手册原文）

1. **联名字段对齐官方模板**。手册 1.2 / 附则 3.2 规定联名人须提供：联名事项、本人签名、联系电话、房号、投票权数、产权证明复印件、**身份证复印件**。v2 在登记表中新增了**身份证照片**上传，字段名也改为官方联名表（附件2-1）的措辞：业主姓名 / 楼栋房号 / 房屋建筑面积 / 联系方式 / 业主本人签名。
2. **投票权数 = 专有部分建筑面积（㎡）**。官方申请书（附件2）明确"业主专有部分总建筑面积（总投票权数）为＿＿平方米"，印证 v1 的双口径模型。
3. **导出严格套用官方模板**。「导出」页生成的申请书逐字对应附件2（含两个勾选框、达标自动打勾），联名表对应附件2-1（表头街道/社区/小区/联名时间、官方列、必附证件备注），并新增**签名预览**（可长截图作签名页）。
4. **新增围绕手册的业主向功能**：成立业委会全流程路线图、资格自测、换物业指南、名词解释、官方文书库（见下）。

## 二、功能一览（四个 tab）

- **进度**：小区名 + 双 20% 口径进度（人数 / 建筑面积，达标取其一）+ 迷你全流程路线图 + 我要联名。
- **联名（核心）**：按官方表格登记、上传产权证与身份证、手写签名；一人一签、可改可撤；发起人审核通过后才计入。
- **指南**：
  - **全流程路线图**（手册一、(一)~(六)）：① 满足启动条件·业主联名 → ② 提交申请·街道办受理 → ③ 成立筹备组(7–10人) → ④ 业主代表/委员报名遴选 → ⑤ 召开首次业主大会·选举业委会 → ⑥ 备案·刻章·开始运作。每步可展开看要点与对应官方文书，并按「当前阶段」高亮。
  - **资格自测**：对照手册 3.3 正面清单（7 条）/ 3.4 负面清单（11 条），报名前自测能否当代表、委员。
  - **换物业指南**：手册 2.22 选聘物业服务企业的招标主体、5 步流程、表决比例（双 2/3 参与 + 双过半同意）。
  - **名词解释**：联名 / 投票权数 / 专有部分 / 两个 20% / 筹备组 / 业主大会与业委会 / 表决比例 / 公示（≥10日）/ 征集意见。
  - **官方文书库**：手册附件 1–30 份示范文本，按阶段归类的导航索引。
- **我的**：我的联名状态、领取发起人权限、（发起人）审核 / 导出 / 小区参数、openid。

## 三、技术架构

- **前端**：原生微信小程序。
- **后端**：微信云开发（云数据库 + 云存储 + 云函数）。个人免费额度足以支撑单个小区。
- **安全模型**：三个集合（`config` / `signatures` / `admins`）全部设为"所有用户不可读写"，客户端不直接碰库，所有读写经唯一云函数 `yezhuApi`（动作路由）。服务端强制：一个微信号一份联名（按 openid 去重）、审核通过才计入分母、审核/导出/改参数校验管理员身份。

## 五、部署步骤

1. **注册小程序拿 AppID**：微信公众平台注册"小程序"，复制 AppID。
2. **导入项目**：微信开发者工具「导入项目」选 `yezhu2` 文件夹，填 AppID。
3. **开通云开发**：点「云开发」新建环境，复制**环境 ID**，改 `miniprogram/app.js` 里的 `envId`。
4. **建三个集合**：云开发控制台新建 `config`、`signatures`、`admins`，权限均设为自定义 `{ "read": false, "write": false }`（仅云函数可访问）。
5. **部署云函数**：右键 `cloudfunctions/yezhuApi` →「上传并部署：云端安装依赖」。
6. **领取管理员**：真机/预览打开 → 「我的 → 领取发起人权限」，首位领取者成为管理员。
7. **设置小区参数**：「我的 → 小区参数」，填小区名、街道办、社区、人数口径分母、建筑面积合计、联名事项、当前阶段、联系人。
8. **推广**：把小程序码发业主群，业主自助「联名」。随进展在「小区参数」更新"当前阶段"，路线图会同步高亮。

## 六、两条 20% 口径

- 人数口径 = 已通过联名户数 ÷ 业主总人数（或户数）
- 投票权口径 = 已通过联名建筑面积合计 ÷ 全体业主专有部分建筑面积合计
- **达标 = 任意一条 ≥ 20%**（对应条例"或者"）。

## 七、合规与数据要点

- **真实性靠审核**：联名人须上传产权证 + 身份证，发起人核验后通过，方计入——这是材料被街道办采信的关键。审核留痕（审核人 + 时间）。
- **房号去重**：不同微信号填相同房号时，审核列表与详情标红提示。
- **共有人**：一套房多个共有人时，建议每户一名共有人提交（计 1 户、1 份面积）；若街道要求全体共有人签字，可线下补签。
- **隐私**：产权证、身份证、手机号仅发起人在审核端可见；普通业主只能看到自己的记录与全局进度。
- **电子签名**：是否接受纯电子汇总件由街道办掌握，建议提交前确认。导出页同时支持复制文本与签名长截图，纸质 / 电子两条路都留好。

## 八、可选增强

- 用云函数 + 模板把申请书、联名表直接生成 Word/PDF 下载。
- 「指南」各官方文书可链接到对应 PDF 页或上传到云存储供下载。
- 新增审核员：在 `admins` 集合加 openid，或给 `yezhuApi` 增加 `addAdmin`（仅现有管理员可用）。
